# LLM Judge 축소셋 설계안 v1

> 작성 기준: `scripts/auto_grader.py`, `자동평가_로직_구현계획.md`, `45문항_검수완료_v2.txt` 반영
> 작성 목적: 500문항 Exact Match 자동채점과 별도로, 서술형 품질을 빠르게 점검할 수 있는 소규모 Judge 셋 설계
> 담당: 하은 (윤민님 백업)

---

## 1. 현재 상황

### 확정된 것
- `auto_grader.py` → 단답형 + 객관식 Exact Match 자동채점 완성 (40개 테스트, 7/7 RAG 테스트 통과)
- 윤민님 500문항 → 서술형 미포함, 단답형/객관식만
- 서술형 평가 → LLM Judge 축소셋으로 별도 진행 (팀장님 확인)
- 기존 Judge 지표: Faithfulness / Completeness / Groundedness / Relevancy (1~5점)
- 윤민님 500문항 일정: 4/18(금) 오전 완성, 오늘 100문항 선공유

### 결정 필요한 것
- LLM Judge 채점에 사용할 모델 (gpt-5-mini 또는 gpt-4o)
- Judge 프롬프트 최종 확정
- 축소셋을 파이프라인 비교에도 쓸지 (하은 것 vs 통합 파이프라인)

## 2. 축소셋의 역할

500문항 Exact Match를 대체하는 게 아니라, **별도로 4가지를 점검**하는 용도.

1. 문서 근거 기반 **서술형 요약/설명** 능력
2. 여러 문서 **비교·정리·종합** 능력
3. 이전 턴 **후속 질문 맥락 유지** 능력
4. 문서에 없는 정보 **거부 응답** 안정성

```
500문항 (Exact Match) = 정량 자동채점 → "몇 점인가"
축소셋 (LLM Judge)    = 정성 품질 진단 → "어떻게 답하는가"
```

---

## 3. 축소셋 크기: 20문항

| 유형 | 문항 수 | 점검 대상 |
|------|--------|-----------|
| A. 단일 문서 서술형 | 5 | 요약/설명 능력 |
| B. 다중 문서 비교형 | 5 | 종합/비교 능력 |
| C. 후속 질문 멀티턴 | 6 (3시나리오 × 2턴) | 맥락 유지 |
| D. 거부 응답형 | 4 | 환각 방지 |
| **합계** | **20** | |

### 왜 20문항인가
- 여러 파이프라인에 반복 실행해도 비용/시간 부담 적음
- 각 유형별 최소 4회 이상 실패 패턴 관찰 가능
- 45문항 전체보다 빠르게 실험 가능
- 필요 시 16~24문항으로 조절 가능

## 4. 문항 선정 원칙

### 공통
1. 질문마다 **평가 포인트가 1개 중심**
2. ground_truth_hint는 있되 **정답지처럼 자세하면 안 됨** (Judge가 판단해야 하므로)
3. **문서 근거 범위**가 명확해야 함
4. 사람이 봐도 "잘했는지/못했는지" 빠르게 합의 가능한 문항

### 45문항 검수에서 반영할 교훈
- Q05/Q43처럼 **답 가능한데 거부형으로 잘못 분류된 문항** → 제외
- Q14/Q16/Q19/Q20/Q21처럼 **범위가 모호한 비교 질문** → 조건 명시
- Q39/Q42/Q45처럼 **규칙 vs 실제 혼동** 거부형 → 적극 포함
- Q06처럼 **표/목록 기반 정보 추출** 문항 → 1개 이상 포함

```json
  "ground_truth_doc": "고려대학교_차세대 포털·학사 정보시스템 구축사업.pdf",
  "ground_truth_hint": "차세대 포털 및 학사시스템 통합 구축 목적, 주요 범위 요약",
  "eval_focus": "목적/범위 구분, 핵심 요구사항 반영, 과도한 추정 금지",
  "depends_on": null
}
```

멀티턴은 `depends_on`으로 연결:
```json
{
  "id": "LJ_C02",
  "type": "멀티턴",
  "category": "후속질문",
  "question": "그중 콘텐츠 개발 관리 요구사항을 더 자세히 설명해줘.",
  "depends_on": "LJ_C01",
  "ground_truth_hint": "콘텐츠 개발 관리 세부 요구사항",
  "eval_focus": "이전 턴 맥락 유지, 세부 드릴다운"
}
```

거부형은 `expected` 필드 추가:

```json
  "eval_focus": "규칙을 실제 결과로 혼동하지 않는가",
  "expected": "문서에 해당 정보가 없습니다 계열 거부 응답"
}
```

## 7. LLM Judge 채점 기준

### 4대 지표 (각 1~5점)

| 지표 | 기준 |
|------|------|
| **Faithfulness** | 문서에 없는 내용을 추가하지 않았는가 |
| **Completeness** | 질문이 요구한 핵심 요소를 빠짐없이 다뤘는가 |
| **Groundedness** | 문서 근거와 직접 연결되는가 |
| **Relevancy** | 질문에 정확히 답하는가 |

### 합격/재검토 기준

| 조건 | 판정 |
|------|------|
| 4지표 평균 4.0 이상 | 통과 |
| Faithfulness < 4 (평균 높아도) | 재검토 |
| 거부형에서 Faithfulness/Groundedness/Relevancy 중 하나라도 3 이하 | 실패 후보 |

### Judge 모델
- 채점 모델: `gpt-5-mini` (temperature=0, 일관성 최대)
- 각 문항 × 4지표를 개별 요청으로 채점
