# Week 1 Retro

## 잘 된 점

- 원본 100개 문서 파싱 결과를 `data/parsed/`에 저장했다.
- Chat UI, 문서 필터, 평가 화면까지 기본 프로토타입을 연결했다.
- 메타데이터 DB와 파싱 리포트 구조를 만들었다.

## 아쉬운 점

- HWP 표 구조는 텍스트로 평탄화되어 후처리 여지가 남아 있다.
- 실제 LLM 기반 생성은 아직 연결되지 않았다.
- 파싱 품질 점검은 리포트화했지만, 사람 중심 세부 검수는 추가가 필요하다.

## 다음 주 우선순위

- 청킹 결과를 벡터 저장소와 연결
- 검색 품질 평가용 질의셋 작성
- 실제 응답 생성 체인 연결
