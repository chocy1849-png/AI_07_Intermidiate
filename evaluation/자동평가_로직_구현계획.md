## 2. 입력 — JSON 포맷 (확정)

윤민님 4/14 확정. `PartA_RFP_AutoGrading_QBank.json`

```json
{
  "dataset_name": "PartA_RFP_AutoGrading_QBank",
  "version": "v1",
  "items": [
    {
      "id": "DOC001_Q001",
      "document_id": "DOC001",
      "document_name": "(사)벤처기업협회_2024년 벤처확인종합관리시스템 기능 고도화 용역사업.hwp",
      "question_type": "short_answer",
      "difficulty": "basic",
      "category": "common",
      "question": "해당 사업의 총 예산은 얼마인가?",
      "answer": "352,000,000원",
      "answer_format": "string",
      "evidence": {
        "page": 4,
        "section": "2. 사업개요",
        "quote": "□ (소요예산) 352,000,000원(부가가치세 포함)"
      }
    }
  ]
}
```

### 문항 유형
- `short_answer` — 단답형 (예산, 기간, 기관명 등)
- `multiple_choice` — 4지선다 (choices 필드 포함, 정답은 보기 텍스트 기준)

### 난이도 구성
- basic 30% — 예산, 기간, 기관
- intermediate 50% — 계약 방식, 평가 기준, 제출 요건
- advanced 20% — 기능 요구사항, 특수 조건

### 정답 표준화 규칙 (윤민님 정의)
- 금액: `000,000,000원` (콤마 포함)
- 날짜: `YYYY-MM-DD`
- 기간: 문서 표현 그대로
- 기관명: 풀네임 유지

## 3. 채점 엔진 설계 (협의 완료)

### 3-1. 채점 규칙 — 팀장님 4/14 확정

| 항목 | 결정 | 근거 |
|------|------|------|
| **공백 처리** | 채점 시 공백 제거 후 비교 | 500문항 작성 시 공백 일일이 통일하는 부담 > 코드 1줄 추가 비용 |
| **객관식** | 텍스트 답변 유도 + 번호 폴백 매핑 | "2번", "2", "두번째" 등 → choices에서 자동 매핑. 형식 때문에 정답을 오답 처리하면 평가 신뢰도 저하 |
| **숫자 표현** | 정확한 금액이면 표현 차이 허용, "약/대략"은 오답 | "352,000,000원" = "3억 5,200만원" (OK). "약 3억 5천만원" (NG). 부정확한 표현은 사용자 관점에서도 좋지 않음 |
| **기본 채점** | Exact Match (부분 점수 없음) | 정답이 하나로 명확한 문항만 출제하므로 Exact Match 충분. 부분 점수 넣으면 기준이 주관적 |
| **베이스 코드** | `04_베이스라인_평가.py` 재량껏 수정 사용 | 팀장님: "사용하기 편한 방식으로 바꿔서 만들고 공유" |

### 3-2. 채점 함수 설계

```python
import re

def 공백제거(text: str) -> str:
    return text.strip().replace(" ", "")

def 숫자정규화(text: str) -> int | None:
    """
    '352,000,000원' → 352000000
    '3억 5,200만원' → 352000000
    '약 3억' → None (약/대략 포함 시 정규화 거부 → 오답)
    """
    if re.search(r"약|대략|대충|정도|추정", text):
        return None
    # 콤마/원 제거 후 숫자 추출, 억/만 단위 변환 등
    ...

def 객관식매핑(model_answer: str, choices: list[str]) -> str:
    """
    '2번' → choices[1]의 텍스트
    '2' → choices[1]의 텍스트
    '두번째' → choices[1]의 텍스트
    """
    번호패턴 = {"1": 0, "2": 1, "3": 2, "4": 3,
               "1번": 0, "2번": 1, "3번": 2, "4번": 3,
               "첫번째": 0, "두번째": 1, "세번째": 2, "네번째": 3}
    cleaned = model_answer.strip()
    if cleaned in 번호패턴:
        idx = 번호패턴[cleaned]
        # choices에서 "1. 제한경쟁입찰" → "제한경쟁입찰" 추출
        return re.sub(r"^\d+\.\s*", "", choices[idx])
    return cleaned

def 채점(model_answer: str, item: dict) -> bool:
    truth = item["answer"]
    qtype = item["question_type"]

    # 1. 객관식 번호 폴백
    if qtype == "multiple_choice" and "choices" in item:
        model_answer = 객관식매핑(model_answer, item["choices"])

    # 2. 공백 제거
    pred = 공백제거(model_answer)
    gt = 공백제거(truth)

    # 3. 문자열 비교
    if pred == gt:
        return True

    # 4. 숫자 정규화 비교 (금액 표현 차이 허용)
    pred_num = 숫자정규화(model_answer)
    gt_num = 숫자정규화(truth)
    if pred_num is not None and gt_num is not None:
        return pred_num == gt_num

    return False
```

### 3-3. 추후 보완 (팀장님 제안)

- 답변 양상 통계를 내서, 새로운 폴백 패턴이 발견되면 지속 추가
- 예: "세 번째", "three" 등 예상 못한 패턴
